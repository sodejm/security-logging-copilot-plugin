# Project Constitution: Security Logging Advisor

This constitution defines the non-negotiable guidelines, architectural constraints, and behavior standards for all development and AI agent operations in this repository. All code changes and feature plans must strictly respect these rules.

---

## 1. SDD (Spec-Driven Development) Core Workflow
- **Specify First**: No code shall be modified or implemented without first creating or updating a feature specification under the `specs/` directory.
- **Continuous Alignment**: Maintain the `project-context.md` file to reflect any architectural shifts or added dependencies immediately following implementation.
- **Commit Standards**: All commits must adhere strictly to Conventional Commits:
  - Format: `<type>(<scope>): <short description>`
  - Types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`.
- **Review and Approval**: AI agents must pause and obtain human approval before starting code modification steps on approved plans.

---

## 2. Licensing and Distribution Principles
- **License Terms**: This project is licensed under the **PolyForm Noncommercial License 1.0.0**. 
- **Enterprise Restrictions**: Enterprises may use this plugin internally for repository scanning, security telemetry, and recommendation generation. However, **baking this plugin or any derivative work into a commercial product or selling it is strictly prohibited**.
- **Attribution**: All distributions and derivative forks must preserve the original copyright notice and attribution to the author.

---

## 3. Coding and Quality Standards
- **Strict Typing**: TypeScript code must use strict type settings. The `any` keyword is disallowed; use `unknown` or concrete interfaces.
- **Python Quality**: Python scripts must follow PEP 8 styling, utilize clear type hints where applicable, and maintain descriptive error handling.
- **Test Alignment**: Any code changes affecting logical scripts (scanning/validation) must update or create corresponding automated validation tests.

---

## 4. Security & Logging Guidelines
- **Zero Secrets Leakage**: Under no circumstances should access keys, tokens, credentials, API endpoints with secrets, or PII be committed to repository code, configuration, or generated report files.
- **Data Redaction**: Structured log configuration recommendations generated by this plugin must implement standard redaction paths for authorization headers, cookies, and user credentials.
- **Log Context Requirements**: Security-relevant logs must capture:
  - Timestamp (ISO 8601 UTC format).
  - Event category (Authentication, Authorization, Data Access, configuration).
  - Anonymized/hashed actor IDs.
  - Action, Target, and Outcome (Success/Failure/Error).
